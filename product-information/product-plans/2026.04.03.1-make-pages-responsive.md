# Plan: Make Pages Responsive — 2026-04-03

## Overview

This plan covers three implementation phases:

1. **`PlatformService`** — a root-level singleton that detects the current device category (phone, tablet, laptop, desktop) using `matchMedia` and exposes reactive signals. Used for conditional rendering and CSS class application across all layouts.
2. **`HamburgerMenuComponent`** — a new shared component (`shared/components/hamburger-menu/`) that owns the hamburger toggle button, the transparent backdrop, and the dropdown menu. It is self-contained: services, state, and all menu items live here. Imported by `HeaderComponent`.
3. **`HeaderComponent` responsive expansion** — injects `PlatformService` and switches between the desktop layout and a slim mobile top bar that composes `HamburgerMenuComponent`. No page layouts need to change.
4. **Responsive layout pass** — each page layout binds a CSS class driven by `PlatformService`. Responsive styles target that class rather than using `@media` queries.

> **Fallback note:** If the platform-service class approach turns out to cause issues (e.g., SSR, testing), the fallback is to define SCSS breakpoint variables in `src/app/styles/_breakpoints.scss` and use conventional `@media` queries pointing at those variables. See the appendix at the bottom of this plan.

## Step 1: Create `PlatformService`

**File:** `src/app/services/platform.service.ts`

### Breakpoints

| Platform   | Width range         |
|------------|---------------------|
| `phone`    | < 768 px            |
| `tablet`   | 768 px – 1023 px    |
| `laptop`   | 1024 px – 1439 px   |
| `desktop`  | ≥ 1440 px           |

### Implementation

```typescript
import { computed, Injectable, signal } from '@angular/core';

export type DevicePlatform = 'phone' | 'tablet' | 'laptop' | 'desktop';

@Injectable({ providedIn: 'root' })
export class PlatformService {

  private readonly phoneQuery  = window.matchMedia('(max-width: 767px)');
  private readonly tabletQuery = window.matchMedia('(min-width: 768px) and (max-width: 1023px)');
  private readonly laptopQuery = window.matchMedia('(min-width: 1024px) and (max-width: 1439px)');

  private readonly _platform = signal<DevicePlatform>(this.detectPlatform());

  readonly platform      = this._platform.asReadonly();

  readonly isPhone       = computed(() => this._platform() === 'phone');
  readonly isTablet      = computed(() => this._platform() === 'tablet');
  readonly isLaptop      = computed(() => this._platform() === 'laptop');
  readonly isDesktop     = computed(() => this._platform() === 'desktop');

  /** True when the viewport is phone or tablet width. */
  readonly isMobile      = computed(() => this.isPhone() || this.isTablet());
  /** True when the viewport is laptop or desktop width. */
  readonly isLargeScreen = computed(() => this.isLaptop() || this.isDesktop());

  constructor() {
    const update = () => this._platform.set(this.detectPlatform());
    this.phoneQuery.addEventListener('change',  update);
    this.tabletQuery.addEventListener('change', update);
    this.laptopQuery.addEventListener('change', update);
  }

  private detectPlatform(): DevicePlatform {
    if (this.phoneQuery.matches)  return 'phone';
    if (this.tabletQuery.matches) return 'tablet';
    if (this.laptopQuery.matches) return 'laptop';
    return 'desktop';
  }
}
```

**Notes:**
- Uses `matchMedia` listeners instead of a `resize` event — fires exactly at breakpoint boundaries, not on every pixel. This is the same standard Web API that CSS itself uses internally, and is already used by `ThemeService` in this project for `prefers-color-scheme`.
- `providedIn: 'root'` means one instance for the lifetime of the app; listener cleanup is unnecessary.

## Step 2: Create `HamburgerMenuComponent`

**Location:** `src/app/shared/components/hamburger-menu/`

### Responsibility split

`HamburgerMenuComponent` is a **dumb shell** — it knows nothing about services or application logic. Its only job is to manage open/closed state and render whatever content is projected into it.

| What lives here | What lives in `HeaderComponent` |
|---|---|
| Hamburger / X toggle button | All `inject()` calls |
| `isMenuOpen` signal | Username display |
| Transparent backdrop | Action buttons (Add Panel, Clear Dashboard, Logout) |
| `<ng-content>` slot | Menu item styles |
| `close()` public method | Calling `close()` after each action |

The parent accesses `close()` through a template reference variable (`#hamburger`) in the header template.

### TypeScript

```typescript
import { Component, signal } from '@angular/core';
import { LucideAngularModule, Menu, X } from 'lucide-angular';

@Component({
  selector: 'app-hamburger-menu',
  templateUrl: './hamburger-menu.component.html',
  styleUrls: ['./hamburger-menu.component.scss'],
  imports: [LucideAngularModule],
})
export class HamburgerMenuComponent {

  readonly menu = Menu;
  readonly x    = X;

  readonly isMenuOpen = signal<boolean>(false);

  toggleMenu(): void {
    this.isMenuOpen.update(open => !open);
  }

  close(): void {
    this.isMenuOpen.set(false);
  }
}
```

### Template

```html
<!-- Hamburger / X toggle button -->
<button class="hamburger-btn" (click)="toggleMenu()" title="Menu">
  @if (isMenuOpen()) {
    <lucide-icon [img]="x" [size]="22"></lucide-icon>
  } @else {
    <lucide-icon [img]="menu" [size]="22"></lucide-icon>
  }
</button>

@if (isMenuOpen()) {
  <!-- Transparent full-screen backdrop — click to close.
       z-index 99 sits below the dropdown (100) but above everything else. -->
  <div class="hamburger-backdrop" (click)="close()"></div>

  <div class="hamburger-dropdown">
    <ng-content></ng-content>
  </div>
}
```

### Styles

`hamburger-menu.component.scss` only styles the shell. Menu item styles live in `header.component.scss` because projected content is styled by the projecting component under Angular's view encapsulation rules.

## Step 3: Update `HeaderComponent` with Mobile Layout

`HeaderComponent` remains the smart component — it owns all service injections and all application logic. Its mobile template projects menu content into `<app-hamburger-menu>` and uses a template reference variable (`#hamburger`) to call `hamburger.close()` after each action.

### Changes to `header.component.ts`

Add `PlatformService` and `HamburgerMenuComponent`. Remove `Menu`/`X` icons — those now live in `HamburgerMenuComponent`. No `isMenuOpen` state here.

```typescript
import { Component, computed, inject, input, Signal } from "@angular/core";

@Component({
    selector: 'app-header',

    styleUrls: ['./header.component.scss'],

export class HeaderComponent {

    readonly showDashboardSpecificContent = input<boolean>(false);
    readonly onLandingPage                = input<boolean>(false);

    readonly layoutGrid = LayoutGrid;
    readonly moon       = Moon;
    readonly sun        = Sun;
    readonly plus       = Plus;
    readonly minus      = Minus;

    readonly themeService     = inject(ThemeService);
    readonly dashboardService = inject(DashboardService);
    readonly authService      = inject(AuthService);
    readonly platformService  = inject(PlatformService);
    readonly router           = inject(Router);

    readonly theme       = computed<Theme>(() => this.themeService.theme());
    readonly currentUser = computed(() => this.authService.currentUser());

    }
}
```

## Step 4: Responsive Layout Fixes via Platform Service Classes

Each page layout component:
1. Injects `PlatformService`
2. Binds `[class.is-mobile]="platformService.isMobile()"` on its root element
3. All responsive styles are written as `.layout-root.is-mobile { ... }` — no `@media` needed

### 4a. Landing Page

**`landing-page-layout.component.ts`** — inject `PlatformService`:

```typescript

```

**`landing-page-layout.component.html`** — add class binding to root div:

```html
<div class="landing-page-layout" [class.is-mobile]="platformService.isMobile()">
    ...
```

## Appendix: SCSS Breakpoint Variables (Fallback)

If the platform-service class approach is ever dropped in favour of conventional media queries, define breakpoints once in a shared partial to avoid magic numbers scattered across files:

**`src/app/styles/_breakpoints.scss`**

```scss
// Import this partial in any component SCSS that needs @media queries:
// @use 'src/app/styles/breakpoints' as bp;
// Then use: @media (max-width: bp.$tablet-max) { ... }

$phone-max:   767px;
$tablet-min:  768px;
$tablet-max:  1023px;
$laptop-min:  1024px;
$laptop-max:  1439px;
$desktop-min: 1440px;
```

Usage example:

```scss
@use 'src/app/styles/breakpoints' as bp;

@media (max-width: bp.$tablet-max) {
    .dashboard-layout-content {
        grid-template-columns: 1fr;
    }
}
```
